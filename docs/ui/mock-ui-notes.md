# モック動画の要点（画面/ナビ/優先順位）

## 参照メモ
- 共有されたモック動画（TEPPEN-MEO-Google-AI-Studio_202601241810_モック録画.mp4）を参考に、代表フレームから画面構成を整理した。
- 動画自体はリポジトリにコミットしない。
- 実装では「左サイドバー + 上部ヘッダー（検索/ユーザーメニュー）」を優先する。

## 画面一覧（モック寄せ）
- ダッシュボード（KPIカード + 推移グラフ + 右側の内訳/サマリ）
- カレンダー（準備中でも導線は配置）
- 投稿（新規/一覧）
- 口コミ・コメント受信箱
- 店舗一覧/店舗詳細
- ユーザー・契約管理（上位権限のみ）

## ナビ構成（優先度順）
- ダッシュボード
- 口コミ・コメント（受信箱）
- 投稿（新規/一覧/テンプレ）
- 店舗
- 初期設定
- カレンダー（準備中）
- 上位権限向け：ユーザー・契約管理 / 運用ツール（診断/ヘルス/ジョブ/監査/リリース）

## 1画面内の優先順位（上→下）
1) 今日やること（次の行動を3件まで）
2) 初めての設定（折りたたみ）
3) 毎日の運用（折りたたみ）
4) 困ったとき（手順書リンク）

## 文言トーン
- 店舗オーナー向けに短く、次の行動が分かる表現
- 技術語は避ける（例: ロケーション/プロバイダ/モック/スタブは不使用）
- できないことは「準備中」「設定が必要です」と明示
