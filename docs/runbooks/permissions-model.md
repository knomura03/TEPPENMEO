# 権限モデル（運用向け）

## 目的
- 権限の意味と責任範囲を誤解なく共有する
- 誰がどの操作を行うべきかを明確にする

## 権限の構造（簡易図）
```
システム管理者（全体）
  └─ 組織（organization）
       └─ メンバー（owner / admin / member / viewer）
```

## 権限の定義
### システム管理者（system admin）
- 影響範囲: **全組織に影響**
- 管理対象: ユーザー管理、監査ログ、診断、リリース準備、実機ヘルスチェック
- 推奨: **運営者はシステム管理者が推奨**

### 組織管理者（org admin / owner）
- 影響範囲: **所属組織のみ**
- 管理対象: ロケーション作成、接続/投稿/レビュー対応、セットアップ進捗

### メンバー / 閲覧者
- 影響範囲: **所属組織のみ**
- 権限: 閲覧中心（投稿/接続などは制限）

## UIでの表記ルール
- 表示は **システム管理者 / 組織管理者** に統一する
- system admin / superuser などの英語表記は表示に使わない

## 運用ルール
- システム管理者の付与/解除は **システム管理者のみ** 実施
- 誤操作防止のため、必ず確認入力（メール再入力＋CONFIRM）を行う
- 監査ログに付与/解除の操作を記録する（機密は記録しない）
