# セットアップ完了チェックの運用ルール

## 目的
- /app/setup の「完了にする」チェックを、実運用で迷わず使えるようにする
- 完了チェックは“運用メモ”として使い、誤解を防ぐ

## 基本方針
- 完了チェックは **証跡ではなく運用メモ** として扱う
- 完了にするのは **owner / admin** を推奨
- 失敗ややり直しがあれば **完了を外して理由を残す**

## いつ完了にするか（推奨）
- Google接続: OAuth接続が成功し、/admin/diagnostics で接続済みになった時
- GBP紐付け: ロケーション詳細でGBP紐付けが完了した時
- Google投稿テスト: 投稿が成功し、投稿履歴で「成功」になった時
- Meta接続: OAuth接続が成功し、/admin/diagnostics で接続済みになった時
- FBページ紐付け: ロケーション詳細でページ紐付けが完了した時
- Meta投稿テスト: 投稿が成功し、投稿履歴で「成功」になった時
- 画像アップロード設定: Storage設定が完了し、画像アップロードが成功した時

## 完了チェックの操作
1) `/app/setup` を開く
2) 該当ステップの **「完了にする」** をONにする
3) 反映できない場合は、画面に表示される理由を確認する

## 失敗時の扱い
- 失敗が判明したら **完了をOFF** に戻す
- 必要に応じてメモ（運用連絡）を残す
- 原因が解消したら再度ONにする

## 注意
- トークン/シークレット/署名URLの完全値は記録しない
- 完了チェックは監査ログの代替ではない
