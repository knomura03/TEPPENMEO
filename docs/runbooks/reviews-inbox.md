# レビュー受信箱（/app/reviews）運用手順

## 目的
- ロケーションを横断してレビュー対応を行い、返信漏れを減らす

## 前提
- ログイン済み
- 返信は owner / admin のみ実行可能
- Googleレビュー返信は GBP 接続・ロケーション紐付けが必要

## クリック手順（一覧の確認）
1) `/app/reviews` を開く
2) フィルタで「未返信のみ」「ロケーション」「期間」を必要に応じて設定
3) **適用** を押して一覧を更新
4) レビューカードの「ロケーション詳細へ」から個別ページへ遷移可能

## 返信手順（Googleのみ）
1) `/app/reviews` を開く
2) Googleレビューのカードを選ぶ
3) 返信欄に本文を入力
4) **返信を送信** を押す
5) 成功メッセージを確認する（返信済みの表示に切り替わる）

## よくある詰まりと次アクション
- **レビューが表示されない**
  - レビュー同期が未実行の可能性 → `/app/locations/<id>` で「レビュー同期」を実行
  - GBPロケーションが未紐付け → 先に紐付けを行う
- **返信が失敗する**
  - 再認可が必要 → Googleを再接続する
  - API未承認 / 権限不足 → GBPのAPI承認とスコープを確認
  - レート制限 → 時間をおいて再実行
- **Metaレビューに返信できない**
  - 仕様上未対応 → Googleレビューのみ返信可能

## モック運用の注意
- `PROVIDER_MOCK_MODE=true` では外部APIを呼びません
- 返信は疑似的に成功し、結果は表示上の確認に留まります

## 注意
- トークン/シークレット/署名URLの完全値は共有・記録しない
