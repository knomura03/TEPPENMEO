# 実機スモークテスト（PROVIDER_MOCK_MODE=false）

値は一切記載せず、クリック/入力単位でのチェックリストだけを記載します。

## 0. 事前確認
- [ ] `pnpm preflight --mode real` を実行し、未設定/未適用がないか確認（値は出ない）
- [ ] `/admin/diagnostics` で環境/マイグレーション警告がないこと
- [ ] テスト用のGoogle/Metaアカウントでログイン可能なブラウザを用意

## 1. Supabase（DB/ストレージ）
- [ ] `supabase db push` または `supabase db reset --linked` を実行済み（データ消失に注意）
- [ ] `/admin/diagnostics` でマイグレーション警告が消えている
- [ ] Storageバケットが作成済み（診断で「バケット：設定済み」）

## 2. Google（GBP）
- [ ] `/app/locations` から対象ロケーションを開く
- [ ] 「Google Business Profile」で接続→同意→接続済み表示
- [ ] GBPロケーション紐付けを実行し、現在の紐付けに表示される
- [ ] 「投稿作成」で本文＋任意で画像（URL/アップロード）を指定し、**Googleに投稿**をONにして送信
- [ ] 「投稿履歴」にGoogleターゲットが追加され、ステータスが成功になる
- [ ] 「レビュー同期」で一覧が更新される
- [ ] 任意のレビューで返信を送信し、「返信済み」表示になる
- 失敗時の見分け方：権限不足/審査待ち/再認可/レート制限の文言を確認し、手順書に沿って再試行

## 3. Meta（Facebook/Instagram）
- [ ] `/app/locations` から対象ロケーションを開く
- [ ] 「Meta」で接続→同意→接続済み表示
- [ ] Facebookページ紐付けを実行
- [ ] 「投稿作成」で本文＋画像（URL/アップロード）を指定し、**Facebookに投稿**をONにして送信
- [ ] （可能なら）Instagram要件を満たして投稿を試す（画像必須）
- [ ] 投稿履歴にMetaターゲットが追加される
- 失敗時の見分け方：permission不足/IG連携条件/再認可を確認

## 4. 口コミ・コメント受信箱
- [ ] `/app/reviews` を開く
- [ ] Google口コミとSNSコメントが表示されることを確認
- [ ] 返信操作ができることを確認
- [ ] 詳細手順は「受信箱（実機確認）」を参照
  - `/docs/runbooks/inbox-real-mode-check`

## 5. Jobs（一括同期・スケジュール）
- [ ] `/app/setup` で「Googleレビューを一括同期」を「今すぐ実行」
- [ ] `/admin/jobs` で最新ジョブが成功になっていることを確認
- [ ] 自動同期をONに設定し、`pnpm jobs:tick` を実行 → `/admin/jobs` に履歴が増える

## 6. 監査ログ
- [ ] `/admin/audit-logs` で期間フィルタを指定し、ログが表示される
- [ ] 詳細を展開し、操作内容を確認（機密が含まれていないこと）
- [ ] CSVエクスポートがダウンロードできることを確認

## 7. 終了条件
- Google：投稿成功＋レビュー同期成功＋返信成功
- Meta：FB投稿成功（IGは条件を満たす場合のみ）
- Jobs：一括同期ジョブが成功、スケジュール設定が保存される
- 監査ログ：操作が記録され、CSV出力できる
