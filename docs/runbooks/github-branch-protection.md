# GitHub main保護設定（手順書）

## 目的
- main への直pushを防止し、必ずPR経由で取り込む運用に固定する

## 手順（クリック単位）
1. GitHubで対象リポジトリを開く
2. 画面上部の「設定」タブを開く
3. 左メニューの「ブランチ設定」を開く
4. 「ブランチ保護ルールを追加」をクリック
5. ブランチ名パターンに `main` を入力
6. 下記の推奨設定にチェックを入れる
7. 「保存」をクリック

## 推奨設定（必須）
- PR必須（レビュー必須）
- ステータスチェック必須（CI）
- 最新コミット必須（マージ前に最新化）
- 直push禁止（保護設定の回避を禁止）

## 推奨設定（任意）
- 会話の解決を必須
- 強制pushを禁止
- ブランチ削除を禁止

## 注意点
- 設定後は main に直接pushできなくなる
- PRテンプレに沿って「変更点/手順/コマンド結果」を記載する
