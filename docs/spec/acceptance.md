# MVP合格基準（最小版）

## 目的
- リリース手前の到達点を明文化し、判断の迷いを減らす

## 合格基準（チェックリスト）

### 起動・認証
- [ ] `pnpm dev` で起動できる
- [ ] `/auth/sign-in` が表示される
- [ ] Supabase設定時にログインできる

### ロケーション管理
- [ ] `/app/locations` で新規作成ができる
- [ ] 作成後に `/app/locations/[id]` に遷移する
- [ ] ロケーション一覧/詳細が表示される

### プロバイダ接続（Google）
- [ ] 接続開始 → コールバック → 接続済みが表示される
- [ ] 再認可が必要な場合に警告が出る
- [ ] 切断ができる

### 管理機能
- [ ] `/admin/diagnostics` が表示できる
- [ ] 必須環境変数の設定有無が確認できる
- [ ] 監査ログが表示できる

### ドキュメント
- [ ] 手動スモークテストのRunbookが最新
- [ ] main保護設定のRunbookが最新
- [ ] 仕様（画面/ユーザー/管理/合格基準）が整備されている

### 品質
- [ ] `pnpm lint` が成功する
- [ ] `pnpm test` が成功する
- [ ] `pnpm typecheck` が成功する
